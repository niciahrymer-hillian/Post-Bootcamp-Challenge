# Contributing

This is a personal learning portfolio, but it follows real team conventions so the habits transfer.

## Commit style
Conventional commits: `feat:`, `fix:`, `docs:`, `refactor:`, `test:`, `ci:`, `chore:`. One feature per commit.

## Branches
`feat/<thing>`, `fix/<thing>`, `docs/<thing>`. Open a PR into `main` even when working solo — it builds the review muscle.

## Each project repo
- Read `docs/LESSON_PLAN.md` before writing code.
- Implement against the `TODO` markers in the skeleton.
- Keep the README and lesson plan in sync as the project evolves.
